REQUIRED ACTION: CONFIGURE EMAIL CREDENTIALS
================================================================================

The system is fully installed but email sending requires SMTP credentials.

OPTION 1: GMAIL (RECOMMENDED)
------------------------------
1. Enable 2-factor authentication on your Google account
2. Go to: https://myaccount.google.com/apppasswords
3. Generate an "App Password" for Mail
4. Edit configuration:

   nano /home/paul/security-scanner/config/email-config.conf

5. Set these values:

   SMTP_USER="nyepaul@gmail.com"
   SMTP_PASS="your-16-character-app-password"
   EMAIL_METHOD="smtp"

6. Save and exit (Ctrl+X, Y, Enter)

OPTION 2: OTHER EMAIL PROVIDERS

================================================================================
SECURITY CHECKS INCLUDED
================================================================================

The scanner performs these comprehensive checks:

 1. Network Services     - Scans for open ports and insecure services
 2. User Accounts        - Checks for empty passwords, multiple root accounts
 3. File Permissions     - Finds world-writable files, SUID/SGID binaries
 4. SSH Configuration    - Reviews SSH security settings
 5. Firewall Status      - Verifies firewall is active
 6. Package Updates      - Detects available security updates
 7. Running Services     - Audits active services
 8. Failed Logins        - Monitors authentication failures
 9. Disk Encryption      - Checks for encrypted volumes
10. Core Dumps           - Reviews core dump configuration
11. Umask Settings       - Validates default file permissions
12. Kernel Version       - Reports current kernel
